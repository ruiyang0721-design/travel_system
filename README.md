# 🧳 智能旅游推荐系统

> 基于 Vue 3 + Django 的智能旅游路线规划平台，融合 K-means 空间聚类算法、暴力穷举与协同过滤，为用户提供个性化行程推荐。

---

## 📖 项目简介

本系统是一个面向旅游爱好者的智能行程规划平台，是本科毕业设计作品。用户只需选择目标城市、游玩天数和兴趣偏好，系统即可自动推荐景点、规划最优路线，并在地图上可视化展示行程轨迹。

### 核心亮点

- **智能路线规划**：标签匹配 + 协同过滤融合推荐 → K-means 空间聚类 + 时间容量均衡 → 暴力穷举路径优化
- **协同过滤推荐**：基于物品的协同过滤（Item-based CF），利用用户收藏数据挖掘景点相似度
- **酒店锚点约束**：支持用户指定酒店或 AI 智能推荐住宿区域
- **拖拽式行程编辑**：用户可自由调整景点顺序，地图实时重绘
- **AI 对话式规划**：接入 DeepSeek 大模型，通过对话收集旅行需求
- **预算精确计算**：自动统计门票费用，辅助旅行预算管理
- **多维度信息**：景点攻略、旅行日记、收藏系统

### 2. 🗺️ 智能路线规划（核心算法）

**算法流程：**

```
用户输入 → 候选景点召回(标签匹配+协同过滤) → K-means 空间聚类
         → 时间容量均衡 → 暴力穷举路径优化 → 可视化展示
```

**Step 1：候选景点召回** (`candidates.py` + `cf.py`)
- 按城市过滤，硬性排除游玩时长 > 8 小时的景点
- 基础评分：`base_score = 0.7 × 标签匹配率 + 0.3 × 评分归一化`
- 协同过滤加分：基于物品的 CF，利用收藏数据计算景点间余弦相似度
- 融合评分：`final_score = 0.6 × base_score + 0.4 × cf_score`
- **时间预算动态截断**：累加景点游玩时长，直到超过 `天数 × 8 小时` 为止

**Step 2：空间聚类 + 时间容量均衡** (`clustering.py`)
- K-means 聚类，K = 游玩天数，每个簇代表一天的行程
- 时间容量均衡：检查每组总时长，超载景点转移到最近的空闲天
- 转移策略：优先踢出离质心最远的景点（对当天路线紧凑性影响最小）

**Step 3：路径优化** (`routing.py`)
- 暴力穷举所有排列，取总距离最短的路线（每天≤5景点，5!=120种排列，毫秒级精确求解）
- 支持酒店锚点闭环约束（酒店 → 景点 → 酒店）
- 欧氏距离计算

**Step 4：酒店智能推荐**
- 方案 A：用户指定酒店 → 使用高德地图获取经纬度
- 方案 B：未指定 → 计算所有候选景点质心作为推荐住宿区域

## 🚀 快速启动

### 环境要求

- Python 3.10+
- Node.js 18+
- MySQL 8.0+
- npm 或 yarn

### 1. 克隆项目

```bash
git clone https://github.com/ruiyang0721-design/travel_system.git
cd travel_system
```

### 2. 配置数据库

```sql
-- 创建数据库
CREATE DATABASE travel_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

修改 `backend/settings.py` 中的数据库配置（用户名、密码、主机、端口）。

### 3. 启动后端

```bash
# 安装 Python 依赖
pip install -r requirements.txt

# 数据库迁移
python manage.py migrate

# 导入景点数据（首次运行）
python backend/api/load_data.py

# 导入攻略数据（可选）
python backend/api/load_tips.py

# 导入收藏数据（可选，用于协同过滤冷启动）
python backend/api/load_favorites.py

# 创建管理员账号（可选）
python manage.py createsuperuser

# 启动后端服务（默认 8000 端口）
python manage.py runserver
```

### 4. 启动前端

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器（默认 5173 端口）
npm run dev
```

### 5. 访问系统

| 地址 | 说明 |
|------|------|
| `http://localhost:5173` | 前端页面 |
| `http://localhost:8000/admin/` | 管理后台 |

## 🧮 核心算法详解

### 候选景点召回 + 协同过滤融合

```python
# 基础评分
base_score = 0.7 × tag_match_ratio + 0.3 × (rating / 5.0)

# 协同过滤（登录用户且有收藏数据时生效）
cf_score = Item-based CF 余弦相似度均值（归一化到 [0,1]）

# 融合评分
final_score = 0.6 × base_score + 0.4 × cf_score

# 时间预算动态截断（非固定数量）
累加景点游玩时长，直到超过 天数 × 8小时
```

- 标签匹配率 = 用户选择的标签与景点标签的交集 / 用户选择的标签数
- 协同过滤：构建用户-景点交互矩阵，计算景点间余弦相似度
- 匿名用户或无收藏数据时，退化为基础评分方案

### K-means 空间聚类 + 时间容量均衡

- 输入：候选景点的 (经度, 纬度) 坐标矩阵
- 聚类数 K = 游玩天数
- 参数：`n_init=10`, `random_state=42`
- **时间容量均衡**：检查每组总时长，超载景点按距离质心从远到近依次转移到最近的空闲天

### 暴力穷举路径优化

```
每天景点数 ≤ 5（受 8 小时时间预算约束）
5! = 120 种全排列，暴力穷举可在毫秒级内求出精确最优解

枚举所有景点排列 → 计算每种排列的总距离 → 取最短路线

有酒店时：路线为 [酒店 → 景点排列 → 酒店] 闭环
无酒店时：直接优化景点排列顺序
```

### AI 对话式规划

```
用户输入自然语言 → DeepSeek 大模型分析
    ↓
提取旅行参数（城市/天数/偏好）
    ↓
参数不完整 → 追问补充
参数完整 → 输出结构化 JSON
    ↓
调用路线规划引擎 → 生成行程
```
