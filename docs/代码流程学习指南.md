## 🗺️ 全局认知：前后端是怎么对话的？

```
┌─────────────────┐         HTTP 请求（URL + 参数）        ┌─────────────────┐
│                 │  ──────────────────────────────────▶  │                 │
│    前端 (Vue)    │                                       │  后端 (Django)   │
│   localhost:5173 │  ◀──────────────────────────────────  │  localhost:8000  │
│                 │         JSON 响应（数据）               │                 │
└─────────────────┘                                       └─────────────────┘
```

**核心流程永远是这个：**
```
用户操作 → 前端发 HTTP 请求 → urls.py 路由 → views.py 处理 → models.py 读写数据库 → 返回 JSON → 前端渲染
```

**关键文件分工：**

| 文件 | 角色 | 类比 |
|------|------|------|
| `urls.py` | 路由表 | 前台接待，把客人带到对应的房间 |
| `views.py` | 业务逻辑 | 各个房间里的工作人员，处理具体事务 |
| `models.py` | 数据库表结构 | 仓库的货架，定义东西怎么放 |
| `serializers.py` | 数据格式转换 | 翻译官，把数据库对象转成 JSON，反过来也行 |
| `frontend/src/views/*.vue` | 页面 | 用户看到的每个页面 |

## 功能一：用户登录（理解认证机制）

> 这是最基础的功能，理解它之后其他需要登录的功能就都懂了。

### 完整流程图

```
用户在登录页输入用户名密码，点"登录"
    │
    ▼
LoginView.vue: handleLogin()
    │
    │  axios.post('http://127.0.0.1:8000/api/login/', {username, password})
    │
    ▼
urls.py: path('api/login/', TokenObtainPairView.as_view())
    │
    │  Django REST Framework 内置的 JWT 登录视图
    │  自动验证用户名密码是否正确
    │
    ▼
验证通过 → 返回 JSON:
    {
        "access": "eyJhbGciOiJIUzI1NiIs...",   ← JWT Token（通行证）
        "refresh": "eyJhbGciOiJIUzI1NiIs..."   ← 刷新用的 Token
    }
    │
    ▼
LoginView.vue: 收到响应后
    │
    │  localStorage.setItem('access_token', response.data.access)  ← 存到浏览器
    │  localStorage.setItem('username', loginForm.value.username)
    │  window.location.href = '/'  ← 跳转首页
    │
    ▼
完成！以后每次需要登录的请求，都在请求头里带上这个 Token
```

### 代码对照

**前端（LoginView.vue）：**
```javascript
const handleLogin = async () => {
    // 1. 校验非空
    if (!loginForm.value.username || !loginForm.value.password) {
        ElMessage.warning('用户名和密码不能为空哦！')
        return
    }
    // 2. 发请求到后端
    const response = await axios.post('http://127.0.0.1:8000/api/login/', loginForm.value)
    // 3. 存储 Token（这就是"登录成功"的本质）
    localStorage.setItem('access_token', response.data.access)
    localStorage.setItem('username', loginForm.value.username)
    // 4. 跳转首页
    window.location.href = '/'
}
```

**后端（views.py - register_user）：**
```python
# 注册时密码加密存储
user = User.objects.create(
    username=username,
    password=make_password(password)  # PBKDF2 加密，不存明文
)
```

### 💡 关键知识点：JWT Token 是什么？

```
普通登录（Session）：服务器记住你是谁 → 占服务器内存
JWT 登录：服务器给你一个加密的"通行证" → 不占内存，每次请求带上就行

通行证里编码了：你是谁（user_id）、什么时候发的、什么时候过期
服务器收到请求 → 解码通行证 → 知道你是谁
```

**后续所有需要登录的接口，都是这样验证的：**
```python
@api_view(['POST'])
@permission_classes([IsAuthenticated])  # ← 这一行：Django 自动检查有没有有效 Token
def save_plan(request):
    user = request.user  # ← 如果 Token 有效，这里自动拿到当前用户
    ...
```

### 完整流程图

```
用户打开首页 http://localhost:5173/
    │
    ▼
HomeView.vue: onMounted() 自动触发
    │
    │  axios.get('http://127.0.0.1:8000/api/spots/', { params: { tag: '', search: '' } })
```

### 代码对照

**前端（HomeView.vue）：**
```javascript
// 页面加载时自动调用
onMounted(() => {
    fetchHotSpots()
})

const fetchHotSpots = async () => {
    const res = await axios.get('http://127.0.0.1:8000/api/spots/', { 
        params: { tag: '', search: '' } 
    })
    allHotSpots.value = res.data.data.slice(0, 8)  // 只取前8个
    hotSpots.value = allHotSpots.value
}
```

**后端（views.py）：**
```python
@api_view(['GET'])  # 只接受 GET 请求
def get_spot_list(request):
    # 从 URL 参数中获取筛选条件
    search_query = request.query_params.get('search', '')
    tag_query = request.query_params.get('tag', '')
    city_query = request.query_params.get('city', '')

    spots = Spot.objects.all()  # 查询所有景点（Django ORM）

    # 逐步过滤
    if city_query:
        spots = spots.filter(city=city_query)
    if search_query:
        spots = spots.filter(name__icontains=search_query)  # icontains = 不区分大小写的模糊匹配
    if tag_query:
        spots = spots.filter(tags__icontains=tag_query)

    # 序列化：把 Spot 对象列表转成 JSON 列表
    serializer = SpotSerializer(spots, many=True)
    return Response({"status": "success", "data": serializer.data})
```

**数据模型（models.py）：**
```python
class Spot(models.Model):
    name = models.CharField(max_length=100)           # 景点名称
    city = models.CharField(max_length=50)             # 所属城市
    price = models.DecimalField(max_digits=8, ...)     # 门票价格
    duration = models.FloatField()                     # 游玩时长(小时)
    longitude = models.FloatField()                    # 经度（地图用）
    latitude = models.FloatField()                     # 纬度（地图用）
    tags = models.CharField(max_length=200)            # 兴趣标签，如"历史,文化,园林"
    rating = models.FloatField()                       # 评分 0-5
```

### 💡 关键知识点：Django ORM

```python
# 你不需要写 SQL，Django 帮你翻译：
Spot.objects.all()                          # SELECT * FROM api_spot
Spot.objects.filter(city="北京")             # SELECT * FROM api_spot WHERE city = '北京'
Spot.objects.filter(name__icontains="故宫")  # SELECT * FROM api_spot WHERE name LIKE '%故宫%'
Spot.objects.get(id=1)                      # SELECT * FROM api_spot WHERE id = 1 （只取一条）

# __icontains 是 Django 的查询语法：
# name__icontains = "故宫"  →  SQL: name LIKE '%故宫%'
# price__gt = 100          →  SQL: price > 100
```

## 功能三：收藏景点（理解需要登录的操作）

> 涉及到"登录状态检查"和"数据库写操作"，是理解认证功能的好例子。

### 完整流程图

```
用户在景点详情页点"收藏景点"按钮
    │
    ▼
SpotDetailView.vue: toggleFav()
    │
    │  先检查有没有 Token：
    │  if (!token) {
    │      ElMessage.warning('请先登录！')
    │      router.push('/login')
    │      return
    │  }
    │
    │  有 Token → 发请求：
    │  axios.post('http://127.0.0.1:8000/api/favorites/toggle/',
    │      { spot_id: 景点ID },
    │      { headers: { Authorization: `Bearer ${token}` } }  ← 带上通行证
    │  )
    │
    ▼
urls.py: path('api/favorites/toggle/', toggle_favorite)
    │
    ▼
views.py: toggle_favorite()
    │
    │  @permission_classes([IsAuthenticated])  ← 先验证 Token 是否有效
    │  无效 → 返回 401 未授权
    │  有效 → 继续执行 ↓
    │
    │  spot = Spot.objects.get(id=spot_id)     ← 查出景点
    │  fav, created = Favorite.objects.get_or_create(user=request.user, spot=spot)
    │                                            ↑ 核心逻辑：
    │                                              如果没收藏过 → 创建记录（created=True）
    │                                              如果已收藏 → 获取到已有记录（created=False）
    │  if not created:
    │      fav.delete()                          ← 已收藏 → 取消收藏（删除记录）
    │      return {"action": "removed"}
    │  return {"action": "added"}                ← 新收藏
    │
    ▼
前端收到响应：
    │
    │  isFavorite.value = res.data.action === 'added'  ← 更新按钮状态
    │  ElMessage.success(res.data.message)              ← 显示提示
    │
    ▼
按钮从 "🤍 收藏景点" 变成 "💖 已收藏"（或反过来）
```

### 代码对照

**前端（SpotDetailView.vue）：**
```javascript
// 页面加载时检查是否已收藏
const checkFav = async () => {
    if (!token) return  // 没登录就不查
    const res = await axios.get(
        `http://127.0.0.1:8000/api/favorites/check/${route.params.id}/`,
        { headers: authHeaders }
    )
    isFavorite.value = res.data.is_favorite
}

// 点击收藏按钮
const toggleFav = async () => {
    if (!token) {
        ElMessage.warning('请先登录！')
        router.push('/login')
        return
    }
    const res = await axios.post(
        'http://127.0.0.1:8000/api/favorites/toggle/',
        { spot_id: parseInt(route.params.id) },
        { headers: authHeaders }
    )
    isFavorite.value = res.data.action === 'added'
    ElMessage.success(res.data.message)
}
```

**后端（views.py）：**
```python
@api_view(['POST'])
@permission_classes([IsAuthenticated])  # ← 必须登录才能用
def toggle_favorite(request):
    spot_id = request.data.get('spot_id')
    spot = Spot.objects.get(id=spot_id)
    
    # get_or_create: 没有就创建，有就获取
    fav, created = Favorite.objects.get_or_create(user=request.user, spot=spot)
    
    if not created:
        fav.delete()  # 已收藏 → 取消
        return Response({"action": "removed", "message": "已取消收藏"})
    return Response({"action": "added", "message": "已收藏"})
```

**数据库模型（models.py）：**
```python
class Favorite(models.Model):
    user = models.ForeignKey(User, on_delete=models.CASCADE)      # 谁收藏的
    spot = models.ForeignKey(Spot, on_delete=models.CASCADE)      # 收藏的哪个景点
    created_at = models.DateTimeField(auto_now_add=True)          # 什么时候收藏的
    
    class Meta:
        unique_together = ('user', 'spot')  # 同一用户不能重复收藏同一景点
```

### 💡 关键知识点：`get_or_create`

```python
# 这是 Django 的一个超实用方法：
fav, created = Favorite.objects.get_or_create(user=user, spot=spot)

# 如果数据库里有 user=A, spot=故宫 的记录：
#   → fav = 那条记录, created = False

# 如果没有：
#   → 自动创建一条, fav = 新记录, created = True

# 相当于：
if Favorite.objects.filter(user=user, spot=spot).exists():
    fav = Favorite.objects.get(user=user, spot=spot)
    created = False
else:
    fav = Favorite.objects.create(user=user, spot=spot)
    created = True
```

## 功能四：智能路线规划（系统核心算法 ⭐）

> 这是整个系统最复杂的功能，涉及算法调用。理解它就理解了整个系统的设计亮点。

### 完整流程图

```
用户在规划页选择：城市="北京"，天数=3，标签=["历史","文化"]
点击"开始规划"
    │
    ▼
PlanView.vue: 组装参数，跳转到结果页
    │
    │  router.push({
    │      name: 'PlanResult',
    │      query: { city: '北京', days: 3, tags: '历史,文化' }
    │  })
    │
    ▼
PlanResultView.vue: onMounted() → fetchPlan()
    │
    │  从 URL 参数中取出 city, days, tags
    │  axios.post('http://127.0.0.1:8000/api/recommend/', {
    │      city: '北京',
    │      days: 3,
    │      tags: ['历史', '文化']
    │  })
    │
    ▼
urls.py: path('api/recommend/', recommend_spots)
    │
    ▼
views.py: recommend_spots()
    │
    │  调用算法引擎：
    │  formatted_itinerary = generate_itinerary_with_hotel(user_data, hotel, user)
    │
    ▼
planner/planner.py: generate_itinerary_with_hotel()  ← 总控函数，调度三个子模块
    │
    │  ╔══════════════════════════════════════════════════════╗
    │  ║  Step 1: 候选景点召回 (candidates.py)                ║
    │  ╠══════════════════════════════════════════════════════╣
    │  ║                                                      ║
    │  ║  1. 查出北京所有景点                                   ║
    │  ║  2. 排除游玩时长 > 8小时的（如迪士尼要玩一整天）        ║
    │  ║  3. 每个景点算基础分：                                 ║
    │  ║     base_score = 0.7 × 标签匹配率 + 0.3 × (评分/5)   ║
    │  ║                                                      ║
    │  ║     例：用户选["历史","文化"]                          ║
    │  ║         故宫标签["历史","文化","皇家"] → 匹配率=2/2=1  ║
    │  ║         base_score = 0.7×1 + 0.3×(4.8/5) = 0.988     ║
    │  ║                                                      ║
    │  ║  4. 如果用户已登录且有收藏数据，计算协同过滤加分：      ║
    │  ║     cf.py: 构建用户-景点矩阵 → 余弦相似度 → CF分数    ║
    │  ║                                                      ║
    │  ║  5. 融合：final = 0.6 × base + 0.4 × cf             ║
    │  ║                                                      ║
    │  ║  6. 按分数排序，累加游玩时长直到超过 3天×8小时=24小时  ║
    │  ║     → 选出约 10-15 个候选景点                         ║
    │  ╚══════════════════════════════════════════════════════╝
    │                          │
    │                          ▼
    │  ╔══════════════════════════════════════════════════════╗
    │  ║  Step 2: 空间聚类 (clustering.py)                    ║
    │  ╠══════════════════════════════════════════════════════╣
    │  ║                                                      ║
    │  ║  1. K-means 聚类：K = 天数 = 3                       ║
    │  ║     输入：每个景点的 [经度, 纬度]                      ║
    │  ║     输出：3 组景点，地理靠近的分在同一天               ║
    │  ║                                                      ║
    │  ║  2. 时间容量均衡：                                     ║
    │  ║     检查每组总时长，如果某天超过 8 小时：              ║
    │  ║     → 把离中心最远的景点转移到最近的空闲天             ║
    │  ║                                                      ║
    │  ╚══════════════════════════════════════════════════════╝
    │                          │
    │                          ▼
    │  ╔══════════════════════════════════════════════════════╗
    │  ║  Step 3: 路径优化 (routing.py)                       ║
    │  ╠══════════════════════════════════════════════════════╣
    │  ║                                                      ║
    │  ║  对每天的景点组，用暴力穷举找最短路线：                 ║
    │  ║                                                      ║
    │  ║  1. 景点数 ≤ 5（受8小时时间预算约束）                  ║
    │  ║  2. 5! = 120 种全排列，毫秒级精确求解                  ║
    │  ║  3. 枚举所有排列，计算每种排列的总距离                  ║
    │  ║  4. 取总距离最短的排列作为最优路线                     ║
    │  ║  5. 有酒店时：[酒店 → 景点排列 → 酒店] 闭环           ║
    │  ║                                                      ║
    │  ╚══════════════════════════════════════════════════════╝
    │
    ▼
views.py: 继续执行
    │
    │  计算预算汇总（门票总计、总时长）
    │  序列化行程数据（Spot 对象 → JSON）
    │  return Response({ "data": {"第1天": [...], "第2天": [...]}, "budget": {...} })
    │
    ▼
PlanResultView.vue: 收到数据
    │
    │  itinerary.value = res.data.data    ← 存行程数据
    │  budgetInfo.value = res.data.budget ← 存预算信息
    │
    │  Vue 模板自动渲染：
    │  - 按天 Tab 切换展示
    │  - 每天的景点时间线列表
    │  - 预算统计卡片
    │  - 地图组件（MapContainer）展示路线
    │
    ▼
用户看到完整的行程规划结果
```

### 💡 关键知识点：算法融合策略

```
为什么不是只用一种算法？

1. 纯标签匹配 → 只考虑兴趣，不考虑位置 → 可能推荐的景点分散在城市两端
2. 纯协同过滤 → 需要大量用户数据 → 新用户/冷启动时没用
3. 纯聚类 → 只考虑距离，不考虑兴趣 → 可能推荐了"近但不感兴趣"的景点

所以系统用"融合"策略：
- 标签匹配（70%）+ 评分（30%）= 基础分 → 保证推荐符合兴趣
- 基础分（60%）+ 协同过滤（40%）= 最终分 → 有数据时更个性化
- K-means 聚类 → 保证每天的景点在地理上紧凑
- 暴力穷举 → 保证每天的游览顺序最省时间（精确最优解）
```

### 💡 关键知识点：Serializer 的 read_only 和 write_only

```python
class TravelDiarySerializer(serializers.ModelSerializer):
    username = serializers.CharField(source='user.username', read_only=True)
    
    class Meta:
        model = TravelDiary
        fields = ['id', 'title', 'city', 'content', 'rating', 'username', 'created_at']
        read_only_fields = ['username']

# read_only=True 的字段：
#   - 只在返回 JSON 时包含（读取时有）
#   - 前端提交时可以不传（写入时忽略）
#
# 效果：
#   GET /api/diaries/ → 返回的 JSON 里有 username（从关联的 user 对象自动提取）
#   POST /api/diaries/create/ → 前端不用传 username，后端用 request.user 自动填充
```

## 功能六：AI 对话式规划（理解流式响应）

> 这个功能比较特殊，用的是 SSE（Server-Sent Events）流式返回，不是普通的 JSON。

### 完整流程图

```
用户在首页 AI 对话框输入："我想去北京玩3天，喜欢历史"
    │
    ▼
HomeView.vue: 发送消息到后端
    │
    │  fetch('http://127.0.0.1:8000/api/ai/chat/', {
    │      method: 'POST',
    │      body: JSON.stringify({ message: "我想去北京玩3天", history: [...] })
    │  })
    │
    ▼
urls.py → views.py: ai_chat()
    │
    │  1. 从用户消息中检测城市关键词
    │  2. 从数据库读取该城市的景点信息，构建 AI 上下文
    │  3. 组装消息列表（System Prompt + 对话历史 + 用户新消息）
    │  4. 调用 DeepSeek API（流式模式）
    │
    │  def stream_generator():
    │      response = client.chat.completions.create(
    │          model="deepseek-chat",
    │          messages=messages,
    │          stream=True,          ← 流式模式：AI 一边生成一边返回
    │      )
    │      for chunk in response:    ← 逐个接收 AI 生成的文字片段
    │          yield f"data: {json.dumps({'type': 'text', 'content': text})}\n\n"
    │
    │  return StreamingHttpResponse(stream_generator(), content_type='text/event-stream')
    │
    ▼
前端逐字接收 AI 回复：
    │
    │  每收到一个文字片段就追加到对话框
    │  效果：用户看到 AI "一个字一个字"地打出回复（像 ChatGPT 一样）
    │
    │  如果 AI 在回复中输出了 JSON 格式的规划参数：
    │  {"action": "plan", "city": "北京", "days": 3, "tags": ["历史"]}
    │  → 前端解析出这个 JSON，显示"生成行程"按钮
    │
    ▼
用户点"生成行程" → 跳转到 PlanResultView → 走正常的规划流程
```

### 💡 关键知识点：SSE 流式响应

```
普通 API：后端处理完 → 一次性返回完整 JSON → 前端一次性显示
SSE 流式：后端一边处理 → 一边推送数据片段 → 前端一边接收一边显示

类比：
- 普通 API = 你点外卖，做好了整份送来
- SSE = 你吃火锅，菜一盘一盘端上来，边涮边吃

为什么 AI 对话要用流式？
- AI 生成一段话需要几秒钟
- 如果等全部生成完再显示，用户要干等好几秒
- 流式模式下，AI 生成第一个字时用户就能看到，体验更好
```

## 🔑 核心概念速查表

### 前端（Vue 3）

| 概念 | 作用 | 代码示例 |
|------|------|----------|
| `ref()` | 声明响应式变量，值变了页面自动更新 | `const count = ref(0)` |
| `onMounted()` | 页面加载时自动执行的函数 | `onMounted(() => { fetchData() })` |
| `v-for` | 循环渲染列表 | `<div v-for="item in list">{{ item }}</div>` |
| `v-if` / `v-show` | 条件渲染（显示/隐藏） | `<div v-if="loading">加载中...</div>` |
| `v-model` | 双向绑定（输入框 ↔ 变量） | `<input v-model="name" />` |
| `@click` | 点击事件 | `<button @click="handleClick">` |
| `:src` | 动态绑定属性 | `<img :src="imageUrl" />` |
| `axios` | 发 HTTP 请求的库 | `axios.get('/api/...')` |
| `router.push()` | 页面跳转 | `router.push('/login')` |
| `localStorage` | 浏览器本地存储（关页面不丢） | `localStorage.setItem('key', 'value')` |

### 后端（Django）

| 概念 | 作用 | 代码示例 |
|------|------|----------|
| `Model` | 定义数据库表 | `class Spot(models.Model): ...` |
| `QuerySet` | 查询数据库的结果集 | `Spot.objects.filter(city="北京")` |
| `Serializer` | 对象 ↔ JSON 转换 | `SpotSerializer(spots, many=True)` |
| `@api_view` | 声明视图函数接受的 HTTP 方法 | `@api_view(['GET', 'POST'])` |
| `@permission_classes` | 声明权限要求 | `@permission_classes([IsAuthenticated])` |
| `request.data` | 获取前端发来的 JSON 数据 | `city = request.data.get('city')` |
| `request.user` | 当前登录的用户对象 | `user = request.user` |
| `Response()` | 返回 JSON 响应 | `return Response({"data": ...})` |
| `ForeignKey` | 外键（表之间的关联） | `user = models.ForeignKey(User, ...)` |
| `unique_together` | 联合唯一约束 | 防止同一用户重复收藏同一景点 |

### HTTP 请求基础

| 概念 | 说明 |
|------|------|
| GET | 获取数据（查），参数在 URL 里 |
| POST | 提交数据（增/改），参数在请求体里 |
| DELETE | 删除数据 |
| URL 参数 | `?city=北京&tag=历史`，用 `request.query_params.get('city')` 获取 |
| 请求体 | JSON 格式，用 `request.data.get('city')` 获取 |
| 请求头 | `Authorization: Bearer eyJhb...`，用于传递 JWT Token |
| 状态码 200 | 成功 |
| 状态码 201 | 创建成功 |
| 状态码 400 | 请求参数错误 |
| 状态码 401 | 未登录/Token 过期 |
| 状态码 404 | 资源不存在 |
